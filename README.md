# Outlook Event Notes

An [Obsidian](https://obsidian.md) plugin that creates notes from Microsoft Outlook
appointments and meetings, capturing the event details (date/time, subject, recipients,
invite message, etc.) using a customisable template.

Drag and drop a meeting from Outlook onto the plugin icon in the Obsidian ribbon. The
plugin will open the existing note for that event if one already exists, or create a new
one if it does not.

By processing `.msg` files directly, the plugin requires no Outlook add-in, no Microsoft
365 admin authorization, and no connection to the Microsoft Graph API.

The plugin relies on the [msgreader](https://github.com/HiraokaHyperTools/msgreader),
[mustache.js](https://github.com/janl/mustache.js), and
[mustache-validator](https://github.com/eliasm307/mustache-validator) libraries.

---

## Installation

Search for **Outlook Event Notes** in Community Plugins, either in the Obsidian app or
on the [Obsidian website](https://obsidian.md/plugins?search=outlook+event+notes).

---

## Usage

Drag and drop a meeting from a calendar in the Outlook Classic desktop app onto the
plugin ribbon icon in Obsidian.

- If a note for that event **already exists**, the plugin opens it.
- If no note exists yet, the plugin **creates one** using your template.

Outlook Classic generates a `.msg` file when you drag and drop an appointment or meeting.
You can also save a meeting as a `.msg` file (or save one received as an email attachment)
and drag and drop the file onto the plugin icon.

Recurring meetings work correctly: the `.msg` file generated by Outlook for a specific
occurrence already contains the correct occurrence date, so the plugin uses it as-is.

## Settings

### Folder location
The folder in which new notes will be created. If the folder does not already exist it
will be created automatically (including any parent folders, e.g. `a/b/c`).

### Filename pattern
The pattern used to name new notes, written in [Mustache syntax](https://mustache.github.io/mustache.5.html).

Default pattern:
```
{{#helper_dateFormat}}{{apptStartWhole}}|YYYY-MM-DD HH.mm{{/helper_dateFormat}} {{subject}}
```
This produces filenames like `2025-03-09 10.31 Discuss documentation`.

You can use `/` in the pattern to create sub-folders automatically:
```
{{#helper_dateFormat}}{{apptStartWhole}}|YYYY/MM MMMM/YYYY-MM-DD HH.mm{{/helper_dateFormat}} {{subject}}
```
This would create `2025/03 March/2025-03-09 10.31 Discuss documentation`.

If `/` characters appear in any Outlook field values, they are replaced by the
**Invalid character substitute** (see below).

### Invalid character substitute
Characters that are invalid in filenames (`/` `*` `"` `\` `<` `>` `:` `|` `?`) are
replaced by this value. Leave blank to remove them entirely.

### Template
The template used when creating new notes, written in Mustache syntax
(see [the manual](https://mustache.github.io/mustache.5.html)). All `.msg`
[fields](https://hiraokahypertools.github.io/msgreader/typedoc/interfaces/MsgReader.FieldsData.html)
are available, plus the helper fields and functions described below.

#### Default template
```
---
title: {{subject}}
subtitle: meeting notes
date: {{#helper_dateFormat}}{{apptStartWhole}}|L LT{{/helper_dateFormat}}
meeting: 'true'
meeting-location: {{apptLocation}}
meeting-recipients:
{{#recipients}}
  - {{name}}
{{/recipients}}
meeting-invite: {{body}}
---
```

YAML values are sanitised automatically to strip characters that would break frontmatter
(`<`, `>`, `*`), and multi-line values are formatted correctly with block scalars.

## Template reference

### Helper fields

#### `helper_currentDT`
The date and time at which the `.msg` file was dropped onto the icon, in ISO 8601 format
(e.g. `2025-04-02T09:31:12+01:00`). Use with `helper_dateFormat` to format it:
```
{{#helper_dateFormat}}{{helper_currentDT}}|YYYY-MM-DD HH.mm.ss{{/helper_dateFormat}}
```

### Helper functions

#### `helper_dateFormat`
Formats a date using [moment.js](https://momentjs.com/). The field name and format string
are separated by `|`.

```
{{#helper_dateFormat}}{{apptStartWhole}}|YYYY-MM-DD HH.mm{{/helper_dateFormat}}
```
→ `2025-03-09 16.14`

```
{{#helper_dateFormat}}{{apptStartWhole}}|L LT{{/helper_dateFormat}}
```
→ locale-aware short date and time (e.g. `09/03/2025 03:19` in English GB,
`03/09/2025 3:19 AM` in English US). The locale follows the Obsidian display language
set in Settings → General → Language.

#### `helper_firstWord`
Returns only the first word of a value. Useful for recording first names only:
```
meeting-recipients:
{{#recipients}}
  - {{#helper_firstWord}}{{name}}{{/helper_firstWord}}
{{/recipients}}
```

## About

This plugin is a fork of [outlook-meeting-notes](https://github.com/davidingerslev/outlook-meeting-notes)
by David Ingerslev, published under the MIT license. Improvements were submitted upstream
but not merged, so this fork is maintained independently.

Key changes from the original:
- Notes for events that already exist are **opened** instead of failing
- The meeting body field is resolved from multiple fallback locations in the `.msg` file,
  improving compatibility with different Outlook configurations
- HTML content in the body field is automatically stripped to plain text
- Recurring event occurrences use the correct date directly from the `.msg` file
- Code cleanup and removal of unused imports
